[ ] JCF*Service 구현체와 File*Service 구현체를 비교하여 공통점과 차이점을 발견해보세요.
    - JCF*Service 구현체와 File*Service 구현체 둘 다 인터페이스를 구현하며, 기능을 제공하고 있다.
    예를 들어 User 구현체에서는 새 유저의 생성, 수정, 삭제, 조회, 전체 조회 등의 CRUD 기능을 제공한다.
    Message, Channel 구현체에서도 이와 같이 해당 기능들을 제공하고 있으므로, '인터페이스의 구현 및 기능의 제공' 이라는 공통점을 가지고 있다고 볼 수 있다.
    즉, JCF* 구현체와 File* 구현체 모두 비즈니스 로직과 저장 로직을 같은 클래스에서 담당한다.

    그러나 JCF*Service 구현체는 일시적이다. 프로그램을 실행하면 백지 상태에서 실행되며, 유저/채널/메시지를 생성, 수정, 삭제 등의 과정을 거친 뒤
    프로그램을 종료한 뒤 다시 프로그램을 실행하면 Map이 초기화되므로, 이전 실행했던 데이터는 남아있지 않다.
    반대로 File*Service 구현체는 지속적으로, 파일에 데이터를 직렬화하여 저장하므로 프로그램을 종료했다가 다시 실행한다 해도 이전의 작업 내용이 사라지지 않는다.

    - create*, update*, delete* 메서드의 경우 각각 중복의 검사, ID 존재의 확인, ID의 삭제 등의 비즈니스 로직을 행한다.

[ ] "저장 로직"과 관련된 코드를 식별해보세요.
    - create*, find*ById, findAll*, update*, delete* 메서드가 전부 새로운 객체의 생성 밎 저장(Map 혹은 File), 순회, 조회하는 저장 로직을 행한다.
    delete* 메서드의 경우 저장소(Map 또는 File)에서 해당 ID or Content를 삭제하는 로직을 행한다고 볼 수 있다.
